# cs2-hub Bolder Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Apply the v0.5 design system from `docs/superpowers/specs/2026-05-05-cs2-hub-bolder-redesign.md` — five-tone palette, 0px corners, no glass — to `cs2-hub/style.css` and `cs2-hub/dashboard.html`. Land it on the dashboard as the proof; remaining pages get separate plans.

**Architecture:** Pure CSS rewrite of shared component classes plus a small markup edit on `dashboard.html` and a small JS edit on `dashboard.js`. Existing token names (`--accent`, `--text`, `--muted`, `--success`, `--danger`, `--warning`, `--special`) are kept as aliases that point at the new palette, so the ~20 places in JS/CSS that reference them keep working without a sweep.

**Tech Stack:** Plain HTML/CSS/JS. No bundler, no test framework for visual changes — verification is "open `cs2-hub/dashboard.html` in a browser."

## Notes Before You Start

**Verification model.** This is a visual change. There are no unit tests for CSS. The verification at the end of each task is: open `cs2-hub/dashboard.html` in a browser and check the listed visual properties.

**Auth.** `dashboard.html` calls `requireAuth()` and redirects to `login.html` if there's no Supabase session. For visual verification you have two options:
- **Real path (preferred):** sign in via `login.html` first, then navigate to `dashboard.html`. You'll see real data and the new styling together.
- **Markup-only path (fallback):** if Supabase is unavailable, comment out `await requireAuth()` in `dashboard.js:11` for the duration of the visual pass. Restore it before any commit. Empty stat cards / empty lists will still show structure and colors.

**Commits.** Commit after each task. Small commits. Don't squash.
